# 로그아웃 리다이렉트 및 /login 제거 대응 계획

## Context
`/login` 페이지를 제거하고 `/` (Home) 페이지에서 로그인을 통합 처리함에 따라, 기존에 `/login`으로 리다이렉트되던 로직(로그아웃 후, 인증 만료 시 등)을 `/`로 수정해야 합니다.

## 계획

### 1. 로그아웃 로직 수정
- **파일**: `src/components/layout/app-sidebar.tsx`
- **변경**: `logout()` 함수 호출 후 `router.push("/")`를 추가하여 명시적으로 홈으로 이동하게 함. (Firebase `signOut` 후 자동으로 페이지가 새로고침되지 않는 경우 대응)

### 2. 하드코딩된 `/login` 경로 수정
- **파일**: `src/app/apps/[id]/page.tsx`
- **변경**: 76번 라인의 `router.push("/?redirect=/apps/${id}")`는 이미 `/`를 사용하고 있으므로 유지. 다만 주석이나 안내 텍스트(214번 라인)에 "로그인 페이지"라고 언급된 부분을 "홈페이지" 또는 "로그인"으로 문구 수정.

### 3. 온보딩 및 보호된 경로 체크 수정
- **파일**: `src/components/providers/onboarding-check.tsx`
- **확인**: 현재 이 컴포넌트는 유저가 있을 때 온보딩이 안 되어 있으면 `/onboarding`으로 보냅니다. 로그아웃 상태에서 `/login`으로 보내는 로직은 `next.config.ts`의 리다이렉트가 처리하지만, 클라이언트 사이드에서 명시적으로 `/login`으로 보내는 코드가 있는지 다시 확인. (현재 Grep 결과로는 없음)

### 4. 기타 확인
- `src/app/page.tsx` (현재의 로그인/홈 페이지)에서 로그아웃 상태일 때의 UI가 적절한지 확인. (이미 확인됨: 로그인 버튼과 최근 앱 목록 표시)

## 검증 계획
1. **로그아웃 테스트**: 대시보드에서 로그아웃 클릭 시 `/`로 정상 이동하는지 확인.
2. **리다이렉트 테스트**: 브라우저 주소창에 직접 `/login` 입력 시 `/`로 리다이렉트되는지 확인 (next.config.ts 설정).
3. **비로그인 접근**: `/dashboard` 등 보호된 경로 접근 시 (만약 미들웨어가 있다면) 어떻게 동작하는지 확인. (현재는 `OnboardingCheck`만 있고 별도의 `AuthGuard` 미들웨어는 보이지 않음. 필요시 추가 검토)

## 배포
- 수정 완료 후 `git commit` 및 `firebase deploy` 실행.
