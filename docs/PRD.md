# GOOTE - Product Requirements Document (PRD)

> Google Play 테스터 관리 대시보드
> 버전: 1.0.0
> 최종 수정: 2024-02-08

---

## 1. 프로젝트 개요

### 1.1 배경
국내 Google Play Console 개발자들이 모인 카카오톡 오픈채팅방에서 상호 테스트를 효율적으로 관리하기 위한 웹 대시보드

### 1.2 목표
- 테스트 요청/참여 프로세스 체계화
- 14일 연속 실행 조건 달성률 향상
- 개발자-테스터 간 상호 협력 문화 구축

### 1.3 핵심 가치
- **상호성**: 모든 유저가 개발자이자 테스터
- **투명성**: 참여 현황 실시간 추적
- **동기부여**: Star 시스템과 랭킹으로 활발한 참여 유도

---

## 2. 사용자 정의

### 2.1 User Roles

| 역할 | 설명 | 권한 |
|------|------|------|
| **User** | 일반 사용자 (개발자/테스터) | 앱 등록, 테스트 참여, 좋아요, 리뷰 |
| **Admin** | 관리자 | 사용자/앱 관리, 공지사항, 통계 열람 |

### 2.2 User Persona

**페르소나 A: 신규 개발자 "민수"**
- 첫 앱 출시 준비 중
- 비공개 테스터 20명 모집 필요
- 테스터 관리 경험 없음

**페르소나 B: 경험 있는 개발자 "지현"**
- 3개 앱 운영 중
- 타 개발자 앱 테스트 참여 활발
- 효율적인 관리 도구 필요

## 3. 기능 명세

### 3.1 인증 (Authentication)

#### 3.1.1 Google 로그인
- **필수 스코프**
  - `openid` - 기본 인증
  - `email` - 이메일 주소
  - `profile` - 프로필 정보 (이름, 사진)
  - `https://www.googleapis.com/auth/gmail.send` - Gmail 발송

- **로그인 플로우**
  ```
  1. 사용자 → "Google로 로그인" 클릭
  2. Google OAuth 팝업
  3. 권한 동의 (최초 1회)
  4. 콜백 → 사용자 정보 저장
  5. 닉네임 설정 (최초 1회, 필수)
  6. 대시보드 이동
  ```

#### 3.1.2 닉네임 설정
- 카카오톡 오픈채팅 닉네임 권장
- **유니크 제약**: 중복 불허
- 프로필 설정에서 변경 가능
- 2~20자, 특수문자 제한

### 3.2 앱 관리 (App Management)

#### 3.2.1 앱 등록

| 필드 | 필수 | 타입 | 설명 |
|------|------|------|------|
| `name` | ✅ | string | 앱 이름 (2~50자) |
| `participationLink` | ✅ | URL | Google Play 테스터 참여 링크 |
| `packageName` | ❌ | string | 패키지명 (com.example.app) |
| `description` | ❌ | string | 앱 소개 (최대 500자) |
| `iconUrl` | ❌ | URL | 앱 아이콘 이미지 |
| `screenshots` | ❌ | URL[] | 스크린샷 (최대 5장) |
| `category` | ❌ | enum | 게임/유틸리티/생산성/기타 |
| `testDuration` | ❌ | number | 테스트 기간 (7~30일, 기본 14일) |
| `minTesters` | ❌ | number | 최소 테스터 수 (기본 20명) |
| `mailTime` | ❌ | string | 메일 발송 시간 (기본 09:30) |
| `mailTemplate` | ❌ | string | 커스텀 메일 내용 |
| `reward` | ❌ | string | 보상 설명 (옵션) |

#### 3.2.2 앱 상태

```
recruiting → testing → completed
                ↓
             deleted (개발자/관리자 요청 시)
```

| 상태 | 설명 |
|------|------|
| `recruiting` | 테스터 모집 중 |
| `testing` | 테스트 진행 중 |
| `completed` | 출시 성공/완료 |
| `deleted` | 삭제됨 (히스토리 보관) |

#### 3.2.3 앱 관리 기능
- 정보 수정
- 모집 마감 (수동/자동/기간)
- 추가 메일 발송 (독려)
- 출시 성공 선언
- 앱 삭제

### 3.3 테스트 참여 (Participation)

#### 3.3.1 참여 플로우

```
1. 테스터 → 앱 목록에서 앱 선택
2. 앱 상세 확인 (소개, 테스트 기간, 보상 등)
3. "테스트 참여" 버튼 클릭
4. 참여 링크로 이동 → Google Play 테스터 등록
5. 대시보드 복귀 → 참여 확인
6. 매일 앱 실행 후 "오늘 실행 완료" 버튼 클릭
7. 14일 연속 달성 → 완료 처리
```

#### 3.3.2 일일 체크
- 하루 1회 "실행 완료" 버튼 클릭
- 자정 기준 리셋 (KST)
- 연속 일수 카운트
- 놓친 날 표시

#### 3.3.3 참여 상태

| 상태 | 설명 |
|------|------|
| `active` | 테스트 진행 중 |
| `completed` | 목표 일수 달성 |
| `abandoned` | 중도 포기 |

#### 3.3.4 중도 포기
- 자유롭게 가능
- 개발자에게 알림 발송
- 패널티 없음

---

### 3.4 메일 시스템 (Email)

#### 3.4.1 메일 발송 주체
- **옵션 A**: 개발자 Gmail 계정
- **옵션 B**: GOOTE 공식 계정
- 개발자가 앱별로 선택 가능

#### 3.4.2 메일 종류

| 타입 | 발송 시점 | 내용 |
|------|----------|------|
| `daily` | 매일 지정 시간 | 앱 실행 독려 |
| `encouragement` | 개발자 수동 발송 | 추가 독려 메시지 |
| `completion` | 출시 성공 시 | 테스트 완료 감사 |

#### 3.4.3 메일 설정
- 기본 발송 시간: 09:30 KST
- 개발자가 변경 가능
- 기본 템플릿 제공 + 커스터마이징

#### 3.4.4 메일 거부
- 테스터가 개별 앱 단위로 수신 거부 가능
- 설정 페이지에서 관리

#### 3.4.5 발송 실패 처리
- 실패 시 개발자에게 알림
- 재시도 없음 (스팸 방지)

---

### 3.5 Star(좋아요) 시스템

#### 3.5.1 기본 개념
- 별점이 아닌 **좋아요(Like)** 개념
- 개수 제한 없음

#### 3.5.2 대상
- **앱에 좋아요**: 테스터 → 앱
- **사용자에게 좋아요**: 개발자 ↔ 테스터 (양방향)

#### 3.5.3 부여 시점
- 언제든지 가능
- 중복 불가 (1인 1좋아요)

#### 3.5.4 리뷰
- 좋아요와 함께 텍스트 리뷰 작성 가능 (선택)
- 최대 200자

---

### 3.6 랭킹 시스템

#### 3.6.1 랭킹 카테고리

| 카테고리 | 기준 | 표시 정보 |
|----------|------|-----------|
| **우수 App** | 좋아요 수 (높은 순) | 앱명, 개발자, 좋아요 수 |
| **다산의 개발자** | 등록 앱 수 + 완료 테스트 수 | 닉네임, 앱 수, 완료 수 |
| **열혈 테스터** | 참여 테스트 수 + 총 연속일 | 닉네임, 참여 수, 총 일수 |

#### 3.6.2 갱신 주기
- 매주 월요일 00:00 KST 갱신
- 이전 주 랭킹 히스토리 보관

#### 3.6.3 표시
- 각 카테고리 상위 10명/10개
- 내 순위 항상 표시

### 3.7 대시보드 화면

#### 3.7.1 홈 (Home)
- 전체 현황 요약
  - 내 개발앱 현황 (모집 중 / 테스트 중 / 완료)
  - 참여 중인 테스트 현황
  - 오늘 실행해야 할 앱 목록
- 최근 알림
- 추천 테스트 앱

#### 3.7.2 내 개발앱 (My Apps)
- 등록한 앱 목록
- 앱별 상세 현황
  - 참여 테스터 수
  - 연속 참여 현황 (테스터별)
  - 좋아요/리뷰
- 앱 등록/수정/삭제

#### 3.7.3 참여 테스트 (My Tests)
- 참여 중인 테스트 목록
- 앱별 연속 참여 현황
- 일일 체크 버튼
- 참여 중단 버튼

#### 3.7.4 앱 탐색 (Explore)
- 테스트 앱 검색/목록
- 정렬: 최신 등록순
- 필터: 참여 가능 여부
- 앱 상세 보기 → 참여 신청

#### 3.7.5 알림 센터 (Notifications)
- 알림 목록
  - 테스터 참여/포기 알림
  - 테스트 완료 알림
  - 메일 발송 실패 알림
  - 좋아요/리뷰 알림
- 읽음/안읽음 표시

#### 3.7.6 통계 (Statistics)
- 내 활동 통계
  - 등록 앱 수
  - 참여 테스트 수
  - 받은 좋아요 수
  - 연속 참여 기록
- 그래프/차트

#### 3.7.7 랭킹 (Rankings)
- 우수 App TOP 10
- 다산의 개발자 TOP 10
- 열혈 테스터 TOP 10
- 내 순위

#### 3.7.8 설정 (Settings)
- 프로필 수정 (닉네임, 사진)
- 메일 수신 설정
- 알림 설정
- 계정 삭제

## 4. 비기능 요구사항

### 4.1 성능
- 페이지 로딩: 3초 이내
- API 응답: 500ms 이내

### 4.2 보안
- HTTPS 필수
- Firebase Security Rules 적용
- XSS/CSRF 방지

### 4.3 접근성
- 반응형 디자인 (모바일 최적화)
- WCAG 2.1 AA 준수

### 4.4 확장성
- 동시 사용자 1,000명 지원
- 앱 등록 10,000개 지원

## 5. 개인정보 처리

### 5.1 수집 정보
- Google 계정 정보 (이메일, 이름, 프로필 사진)
- 닉네임
- 앱 등록/참여 활동 내역

### 5.2 정보 공개 범위
- 닉네임: 전체 공개
- 이메일: 테스트 참여 시 해당 개발자에게만 공개
- 활동 내역: 전체 공개 (랭킹, 통계)

### 5.3 정보 보관
- 계정 삭제 시 개인정보 즉시 삭제
- 활동 내역은 익명화하여 보관 가능

## 6. 릴리즈 계획

### Phase 1: MVP (4주)
- 구글 로그인
- 앱 등록/참여
- 일일 체크
- 기본 대시보드
- 메일 발송

### Phase 2: 확장 (3주)
- Star/좋아요 시스템
- 랭킹 시스템
- 상세 통계
- 알림 센터 고도화

### Phase 3: 고도화 (2주)
- 관리자 기능
- 가이드 페이지
- 성능 최적화
- 버그 수정
